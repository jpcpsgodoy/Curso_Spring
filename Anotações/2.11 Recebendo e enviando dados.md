
# 📡 Enviando e Recebendo Dados com Parâmetros em REST APIs – Spring Boot


Parâmetros são valores adicionais enviados em uma requisição HTTP para:
- Filtrar ou ordenar dados
- Identificar recursos
- Controlar o comportamento da resposta

> Exemplo: escolher o tamanho e os ingredientes ao pedir uma pizza 🍕

---

## 🔄 Tipos de Parâmetros em REST APIs

| Tipo               | Onde aparece                | Usado para...                              |
|--------------------|-----------------------------|---------------------------------------------|
| **Path Parameter** | Na URL                      | Identificar recurso específico (`/users/1`) |
| **Query Parameter**| Após `?` na URL             | Filtro, ordenação, paginação (`?sort=asc`)  |
| **Request Body**   | Corpo da requisição (JSON)  | Enviar dados completos (ex: novo livro)     |
| **Header**         | Cabeçalho HTTP              | Autenticação, tipo de conteúdo, etc         |


## 🧪 Métodos HTTP e envio de dados

### 🔹 `GET`
- Recupera dados
- Parâmetros geralmente via **query string**
- Ex: `/books?category=romance&sort=title`

**Exemplo:**

        
    @RestController
    @RequestMapping("/api")
    public class DataController {
    
        // This method responds to GET requests at /api/data with optional query parameters
        @GetMapping("/data")
        public String getData(@RequestParam(value = "name", defaultValue = "World") String name) {
            // Returns a simple greeting message
            return "Hello, " + name + "!";
        }
    }

### 🔹 `POST`
- Cria um novo recurso
- Dados enviados no **corpo da requisição (JSON)**
- Anotação: `@PostMapping` + `@RequestBody`

**Exemplo:**

    @RestController
    @RequestMapping("/api")
    public class UserController {
    
        // This method handles POST requests at /api/users and expects user data in JSON format
        @PostMapping("/users")
        public String createUser(@RequestBody User user) {
            // Here, you would typically save the user data to a database
            return "User created with name: " + user.getName();
        }
    }




### 🔹 `PUT`
- Atualiza um recurso existente
- Envia dados atualizados no corpo + ID no caminho
- Anotação: `@PutMapping("/{id}")` + `@PathVariable`

**Exemplo:**

    @RestController
    @RequestMapping("/api")
    public class ProductController {
    
        // This method handles PUT requests at /api/products/{id} and expects updated product data
        @PutMapping("/products/{id}")
        public String updateProduct(@PathVariable Long id, @RequestBody Product product) {
            // Here, you would typically update the product information in a database using the id
            return "Product with ID " + id + " updated to name: " + product.getName();
        }
    }

### 🔹 `DELETE`
- Remove um recurso pelo ID
- Anotação: `@DeleteMapping("/{id}")` + `@PathVariable`

**Exemplo:**

    @RestController
    @RequestMapping("/api")
    public class OrderController {
    // This method handles DELETE requests at /api/orders/{id}
	    @DeleteMapping("/orders/{id}")
	    public String deleteOrder(@PathVariable Long id) {
	        // Typically, you would delete the order from a database using the id
	        return "Order with ID " + id + " has been deleted.";
	    }
    }


## ✅ Conclusão

-   Parâmetros tornam as APIs flexíveis e dinâmicas
    
-   Tipos principais: path, query, request body e headers
    
-   Métodos HTTP controlam como os dados são enviados ou recebidos
    
-   Testes com `cURL` ajudam a validar endpoints de forma simples e prática
